# Factory Design Pattern for LWC Components

The purpose of this repo is to demostrate the possibilities of appling the factory patter for Lighting Web Components development

## Why should I use this approach?

1. It will help to keep the object information and feature specific logic in one place
2. Changes are done in one place insted of refactoring greater amout of code
3. The naming convetion is aligned for the whole project
4. It will the minimize amout of redundant code accross the project
5. Maintenace of the code will be much easier

## When not to use it

The creation of records should be handled by uiRecordApi not the factory methods
